Style Guide

ComPAIR is an open-source learning application being developed by the University of British Columbia's Centre for Teaching, Learning & Technology (CTLT) using a GPL-3.0 license. ComPAIR primarily uses default Bootstrap grids and styles. This basic style guide spells out application-specific uses of those styles.

Introduction

Users in ComPAIR interact with three main types of objects in the application: courses, assignments, and users. These objects have slightly different styles applied, as noted below.

Breadcrumbs

Breadcrumbs function as the main navigation for the application and appear just below the global administration bar for every page except the home page. The home page is always the first link, with all child pages listed and linked up until the current page. The current page is listed but not linked.

The breadcrumb labels should match the h1 on each page; these are dynamically generated and found in the compair-config.js file in the static folder, though additional modifications are also made on-the-fly (using breadcrumb.options) in individual controllers to add object names.

Headings

  • h1: Used as first heading on each screen; no content should appear above this heading; should match breadcrumb label; "First Letter Each Word Capitalized" when hard-coded (object h1s are user-generated); object h1s include the icon for that object before the text; no other h1s include icons
    • Courses: <i class="fa fa-book" aria-hidden="true"></i> before course name
    • Assignments: <i class="fa fa-comments" aria-hidden="true"></i> before assignment name
    • Users: <i class="fa fa-user" aria-hidden="true"></i> before user name

Outside UBC

Outside of UBC, you will need to set up your own instance of the application. A ComPAIR instance requires a minimum of a web server running Python 2.7 or 3 and a MySQL or MariaDB database. We also recommend running Redis and a Celery daemon to reduce server load for certain tasks.

  • CentOS 7 Server (Standalone Linux): Set up an all-in-one ComPAIR server. Useful for piloting ComPAIR or small scale production environments.
  • Amazon Web Services (AWS): Set up ComPAIR on Amazon Web Services. Useful for larger scale production environments.
  • Google Container Engine / Kubernetes: Set up ComPAIR on Google Container Engine or Kubernetes. Useful for larger scale production environments.
  • Development Instance via Docker: Set up a development environment for ComPAIR. Useful for contributing to ComPAIR or customizing code. Not recommended for use in a production environment. (Instead, follow the Kubernetes guide above.)
